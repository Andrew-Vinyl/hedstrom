### Structural Spacing & Layout Constraints

When generating documents such as reports, PDF briefs, or slide decks, use the following structural padding and spacing frameworks to match Vinyl's clean layout style.

*   **Slide / Document Edges (Margin)**: Utilize generous outer margins (equivalent to a `1.5rem` to `2rem` web margin) to ensure content breathes and never touches the edge of the canvas.
*   **Vertical Section Pacing**: For transitioning between major visual sections on a long-form document, provide vast vertical whitespace (translating the `5rem` spacing rule from the brand).
*   **Content Backgrounds**: Use palette8 (`#ece8e2`) as a primary document canvas background. For distinct content blocks or inner-panels, use a crisp white (`#ffffff`) overlay block.
*   **Footer/Accent Blocks**: When ending a document or building a stark contrast section, use a deep blue-green (`#006778`) background to anchor the design.

## Design Element Guidelines

The following rules apply when designing specific UI-like elements within documents, such as call-out boxes, tables, and button-like CTAs.

### Call-to-Action (CTA) / Button Elements

Even in PDFs, visual buttons or "click here" elements should follow these strict design rules:
*   **Typography**: JetBrains Mono, Uppercase. Ensure the font size is highly legible but slightly smaller in relative scale to body text.
*   **Color mappings**:
    *   **Primary Background**: Transparent or palette2 (`#215387`).
    *   **Text Color**: palette3 (`#0d5a67`).
*   **Depth & Shadow**: Buttons should *not* have dark, distinct drop shadows by default. Keep the design flat. If a "hover" or active state needs to be simulated visually, apply a very soft, heavily blurred, extended shadow.

### Container Rules (Tables, Data Grids, Content Cards)

*   **Tables and Data Displays**: Keep tabular data clean and uniform. Use DM Sans (the primary body font) and avoid mixing typefaces in tables.
*   **Card Constraints**: When nesting content inside cards (e.g., a "white box" sitting on an `#ece8e2` background), do not let the box stretch arbitrarily wide. If the content is narrow, constrain the box width cleanly rather than stretching it.
